## ğŸ”‘ KEY FRAMEWORKS

### 1. Iterative Clarity Model (10% â†’ 50% â†’ 99%)

**Concept:** Don't try to figure everything out on day 1

**10% Clear (Week 1):**
- Initial direction from kickoff
- Rough ideas to react to

- Clearer roadmap emerging

**99% Clear (Weeks 4-6):**
- Customer research complete
- Final offer locked in
- Detailed 90-day plan
- Confidence to execute

**Why this works:**
- Reduces overwhelm
- Creates fast momentum
- Allows course correction
- Builds confidence through iteration

**Script to explain:**
```
"We're not going to try to nail everything today. We start at 10% clarity - 
enough to move forward. Then we get to 50% over a few weeks. Then we refine 
to 99% before we execute. That way we're never stuck, never overwhelmed, and 
we can course-correct as we learn."
```

**Concept:** Give them something to react to, don't start from scratch

**Script:**

### Analyzing Churn

**Monthly churn review:**
```

2. Group by reason (category breakdown)
3. Identify patterns:

   - No clear pattern? (natural churn, acceptable)

4. Prioritize fixes:
   - HIGH: Affects >30% of churners
   - MEDIUM: Affects 10-30%
   - LOW: One-off issues
```

**Don't try to save everyone.** Some churn is natural.
